What it does

Fifteen slugs across three waves build one product: real focus time drives a virtual road trip up Vietnam. The engine is the honest core; everything else is a faithful view of it.

🧭
Honest core loop
journey-engine — pure framework-free Dart converts active focus into km (speed-only), tracking journey-time vs raw-active-time separately and persisting across sleep/wake & midnight.
🔒
Privacy-clean signal
activity-detection — reads only aggregate system idle-seconds + screen-lock (macOS Swift / Windows C++), never keystrokes or screen. The auditable trust foundation.
🛣️
POV road scene
journey-view → journey-scene-v2 → journey-scene-art-v3 — a Flame first-person road that winds, scrolls slower & calmer, and wears a cohesive hi-res stylized-flat Vietnam art set.
🚗
First-person cockpit
journey-pov + journey-cockpit-lean — a car / motorbike cockpit composited over the road that leans into the curves, bounded & reduce-motion-safe.
🗺️
Real-Vietnam map
route-progress + map-experience — an ordered province chain on real lat/long with OSM tiles, folded into the journey tab, with idle stretches painted red.
🧳
Route authoring
route-planner-v2 — pick any start & end province, mark stops (auto-inserted), review before starting, and abandon a route without ever resetting lifetime distance.
📊
Stats, badges & trust
local-stats — daily/weekly stats with raw-active shown separately from journey time, four badge families, and the onboarding/privacy screen whose claims must match the code.
🪟
Always-on-top presence
mini-window — a frameless always-on-top PiP compact mode plus a tray/menu-bar icon that keeps tracking after the main window is closed.
⏸️
Idle accounting
idle-accounting — counts idle from the moment the state changes (not the next tick) and produces the distance-keyed segment record the red map trace consumes.
🏍️
Choose your vehicle
vehicle-picker — a persisted icon picker (Settings + route-start) that cosmetically overrides the displayed skin (vehiclePreference ?? engineMode) without ever touching accrual.
🌀
Dynamic sweeping curve
journey-dynamic-curve — intensifies the road into F1-style bends that sweep with the scroll phase (no wall-clock), preserving even arc-length object spacing.

Domain Knowledge

Problem statement & solution — from the umbrella spec, journey-engine
ProblemSolution
Vietnam Focus Journey turns real focus time into a virtual road trip up/down Vietnam's province chain. Something has to be the core loop that converts "the user is actively working" into "the traveller moved N kilometres" — and does it honestly, deterministically, and in a way the rest of the app (Flame scene, map, stats) can trust. Today there is nothing between the raw OS signal and what the user sees: distance can't accrue, daily progress can't persist across restarts, and there is no separation between journey time (drives distance, includes the idle grace) and raw active time (drives stats/streaks) — the single most important honesty rule of the product. Deliver that loop as pure, framework-free Dart: an engine taking an injected clock + injected ActivityPlugin, advancing on real elapsed-time deltas, surviving sleep/wake gaps, and persisting/restoring daily progress locally. Exposes distanceKm, journey time, raw active time, idle time, state and cosmetic mode; two-knob active/idle/paused policy; speed-only distance; day-boundary reset at local midnight. Fully unit-testable with no real timers, Flame, or native code.
