# Business rules

Invariants, policies, and constraints that must hold regardless of feature. Each rule should be:
- **Numbered** — so specs and tests can cite it (`BR-12`).
- **Testable** — if you can't write a test for it, it's a goal, not a rule.
- **Owned** — name the person or team accountable for its correctness.
